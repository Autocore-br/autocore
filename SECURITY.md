# Security Policy

## Supported Versions

Atualmente apenas a branch `main` recebe atualizações de segurança.

| Version | Supported |
| ------- | --------- |
| main    | ✅ |

---

## Reporting a Vulnerability

Se você encontrar uma vulnerabilidade de segurança, NÃO abra uma issue pública.

Entre em contato diretamente pelos canais privados do projeto.

Inclua:

- descrição da vulnerabilidade
- impacto esperado
- passos para reproduzir
- possíveis provas de conceito
- sugestões de correção (opcional)

---

## Response Policy

Tentaremos:

- confirmar o problema rapidamente
- reproduzir a vulnerabilidade
- corrigir em tempo razoável
- publicar correções de forma segura

---

## Scope

Esta política cobre:

- backend
- autenticação
- banco de dados
- APIs
- infraestrutura Docker
- pipelines CI/CD

---

## Good Faith

Pesquisadores de segurança que reportarem vulnerabilidades de boa fé não serão penalizados.

Pedimos apenas:

- não explorar dados de usuários
- não causar indisponibilidade
- não divulgar vulnerabilidades antes da correção

## Security Practices

O projeto utiliza:

- criptografia para dados sensíveis
- hashes seguros para senhas
- controle de permissões por roles
- proteção de branches
- revisão obrigatória via Pull Request
